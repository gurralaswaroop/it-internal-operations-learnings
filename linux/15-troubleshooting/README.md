# Linux Troubleshooting Basics

## Introduction
Troubleshooting is the process of diagnosing and resolving issues related to system performance, networking, services, or storage.

## Basic Troubleshooting Steps

Identify the problem

Check system logs

Verify running processes

Check network connectivity

Verify disk and memory usage

## Useful Diagnostic Commands

top – Monitor running processes

free -m – Check memory usage

df -h – Check disk space

ps aux – View running processes

journalctl -xe – View system logs

ping google.com – Test internet connectivity

## Service Troubleshooting

Check service status

systemctl status service-name

Restart service

systemctl restart service-name

## Conclusion
Effective troubleshooting requires understanding logs, system resources, and service behavior.
